# Definición del proceso de desarrollo de PSIREG

VSEST-Lite organiza el trabajo mediante ISO/IEC 29110, Kanban y trazabilidad, y relaciona la evaluación del producto con ISO/IEC 25010 e ISO/IEC 25040. Las fases configuradas son Identificacion, Planificacion, Elaboracion, Revision, Correccion, Validacion y Cierre.

| Actividad | Tareas concretas | Artefactos |
|---|---|---|
| Planificar | Delimitar alcance, responsables, tareas, estados y riesgos. | Plan, issues y tablero. |
| Gestionar requisitos | Revisar RF/RNF, casos de uso y dependencias. | Línea base y matriz maestra. |
| Elaborar | Construir o ajustar modelos, pantallas, arquitectura y documentos. | Artefactos técnicos y documentales. |
| Revisar | Comparar con requisitos y registrar observaciones. | Checklist, comentarios e issues. |
| Corregir y validar | Atender observaciones y repetir verificaciones. | Evidencia y resultado. |
| Verificar SQA | Ejecutar las pruebas aplicables o registrar su limitación. | Reportes o causa de no ejecución. |
| Cerrar | Consolidar evidencia, riesgos y pendientes. | Dashboard y acta de cierre. |

Las columnas válidas del Kanban son Pendiente, En elaboracion, En revision, Con observaciones, Aprobado, Cerrado y No ejecutable.
