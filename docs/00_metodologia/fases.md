# Fases del proceso VSEST-Lite

| Fase | Definición | Evidencia mínima |
|---|---|---|
| Identificación | La tarea, requisito o artefacto fue reconocido dentro del proceso. | Registro inicial en matriz o tablero. |
| Planificación | Se definió responsable, alcance y evidencia esperada. | Entrada en matriz de ejecución. |
| Elaboración | El artefacto está siendo construido. | Archivo creado o avance documentado. |
| Revisión | El artefacto fue completado y está siendo validado. | Observaciones o checklist. |
| Corrección | Existen ajustes pendientes. | Acción correctiva registrada. |
| Validación | El artefacto cumple criterios definidos. | Resultado de revisión. |
| Cierre | El artefacto queda aprobado y vinculado a evidencia final. | Referencia en matriz de trazabilidad. |

## Regla

Cada tarea debe tener una sola fase activa. Si existen observaciones abiertas, no puede pasar a Cierre.
