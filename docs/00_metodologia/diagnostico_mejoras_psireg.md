# Diagnóstico y propuesta de mejoras

El diagnóstico diferencia evidencia de análisis y diseño de evidencia de implementación o prueba. Los hallazgos principales son: versiones documentales inconsistentes, deriva de actores y requisitos, trazabilidad incompleta, ausencia de un flujo uniforme de estados, falta de pruebas ejecutadas, riesgos no consolidados y mezcla entre calidad del proceso y calidad del producto.

| Mejora | Acción |
|---|---|
| M-01 | Mantener una sola versión vigente por sección. |
| M-02 | Utilizar la línea base de 19 RF, 22 RNF y los actores documentados. |
| M-03 | Relacionar requisito, CU, CP, EV y evidencia. |
| M-04 | Registrar cada artefacto con código, fase, estado y próxima acción. |
| M-05 | Ejecutar el Plan SQA únicamente cuando exista el elemento requerido. |
| M-06 | Registrar probabilidad, impacto, prioridad y mitigación de cada riesgo. |
| M-07 | Evaluar el producto con criterios SQuaRE separados del proceso. |
| M-08 | Registrar como No ejecutable una prueba impedida y conservar la causa. |
